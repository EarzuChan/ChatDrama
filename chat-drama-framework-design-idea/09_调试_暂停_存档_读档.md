# 09. 调试、暂停、存档、读档

## 1. 调试原则

调试时允许暂停世界。

进入调试暂停后：

- WorldClock 冻结
- PresentationClock 冻结
- ResponseWindow timer 冻结
- Scheduled bubbles 不继续显现
- LLM response 不继续被消费
- 未进入原子区的请求和响应可编辑

退出调试后：

- 应用修改
- 恢复 timer
- 从当前 stage 继续

---

## 2. 原子区

原子区不可暂停。

包括：

- EventLog append
- PlayerMessage 进入 visible timeline
- NPC bubble 从 staged 变 visible 的瞬间
- StatePatch commit
- Snapshot 最终写入
- 必须连续传递的内部 payload handoff

原则：

```text
原子区必须短。
不可暂停，但可标记 pause_pending。
原子区结束后立刻暂停。
```

---

## 3. LLM 请求调试

### BeforeSend

请求尚未发出。

允许：

- 编辑 request
- early return fake response
- 取消真实发送

### InFlight

请求已发出。

规则：

- 不强制中断 HTTP
- 世界 timer 可冻结
- 响应返回后停在 AfterResponse

### AfterResponse

响应已返回但未消费。

允许：

- 覆写 raw response
- 重 parse
- 替换为 mock response
- 丢弃并重发

## 5. 存档原则

玩家存档比调试暂停更严格。

玩家存档只允许在：

```text
Phase 已显现完成
ResponseWindow 已打开
等待用户输入
```

即用户回应窗期间。

---

## 6. 不允许玩家存档的时机

- Phase composing
- Phase presenting 中
- LLM 请求中
- ActorDraft staged 但未显现
- State commit 中
- Transaction 未关闭

---

## 7. 存档内容

玩家存档保存：

- WorldState
- Chat visible timeline
- 当前 foregroundChat
- 当前 ResponseWindow 定义
- 角色状态
- 角色记忆
- 玩家已知线索
- EventLog offset
- 当前 Phase 已完成状态
- 当前处于“等待用户回应”

不保存：

- 半截 LLM response
- 未显现 NPC 草稿
- staged messages
- debug-only trace 临时对象

---

## 8. 读档规则

读档后进入：

```text
LOADED_WAITING_FOR_USER_FOCUS
```

表现：

- 聊天停在用户回应窗
- timer 不走
- 用户点击输入框后，timer 按完整 window 时长重新开始

不是恢复剩余时间，而是重新给完整回应窗时间。

---

## 9. Trace 与回放

每轮记录：

```text
traceId
playerInput
interpretedInput
phaseBrief
beatPlans
actorInputs
actorOutputs
validatorResults
visibleMessages
statePatches
randomSeed
modelResponseIds
```

要求：

- 支持 replay from trace
- 支持 mock LLM response
- 支持覆写某层输出后继续跑
- 关键 storylet 必须有测试用例
