# 02. 核心概念

## 1. PhoneState

一个存档就是“一台手机”。

```text
PhoneState
  foregroundChatId
  chats[]
  worldState
  characters[]
  eventLog
  snapshots
```

手机内同时可以有任意个聊天会话。处于窗口期且具有“窗口期超时Timer”的聊天会话，当且仅当位于前台且输入框激活时，Timer才会开始计时，离开前台时Timer归零。

---

## 2. ChatSession

聊天会话。

```text
ChatSession
  id
  mode: PRIVATE / GROUP
  participants
  visibleTimeline
  currentPhase
  ResponsePhase
  foregroundStatus
  unreadCount
  pendingBumps
```

### PRIVATE

主角和单个 NPC 的私聊。

### GROUP

主角和多个 NPC 的群聊。

群聊不是多个 Actor 独立抢消息，而是一个 Phase 内多个 Actor 被机制编排。

---

## 3. Phase

一**期**对话。

Phase 是 NPC 侧有目的的一组发言编排。

它回答：

> 这一期 NPC 侧要完成什么？

例如：

- 艾琳承认去过旧车站，但不透露伯爵
- 群聊中三人围绕火灾争执
- 凯抛出怀疑，米娜打圆场

---

## 4. Beat

Beat 是 Phase 内部的一个叙事拍点。

一个 Beat 通常对应：

```text
某个 Actor 在某个任务下生成一个或多个气泡
```

例：

```text
Beat:
  speaker = AILIN
  task = 先防备，反问玩家为什么知道旧车站

输出:
  “……你为什么会知道旧车站？”
  “谁告诉你的？”
```

---

## 5. Transaction

Transaction 是运行时状态提交单位。

它负责：

- 哪些消息变 visible
- 哪些状态 patch 可提交
- 哪些未显现内容被取消
- 哪些 reveal 绑定到具体消息 visible

---

## 6. ResponsePhase

用户回复窗口期。

Phase 显现完成后，系统打开回应窗。

回复窗口期内：

- 用户可以输入多条气泡
- timer 只在前台聊天运行
- 后台聊天回应窗 timer 停止
- 读档后 timer **不自动恢复**，等用户输入框激活后后重新完整计时

---

## 7. WorldState

世界状态。

包括：

- 当前章节
- 主线阶段
- 全局 tag
- 世界事实
- 线索状态
- 路线状态
- 任务状态
- 角色关系

---

## 8. CharacterState

角色状态。

包括：

- 情绪
- 当前目标
- 与玩家关系
- 角色知识
- 角色记忆
- 误解
- blocked memory

---

## 9. Knowledge Isolation

角色认知隔离。

Actor 只能看到自己当前应该知道的内容。

它不能看到：

- 全局真相
- 未来剧情
- 其他角色私密记忆
- 未解锁线索

---

## 10. Storylet

可触发的剧情单元。

Storylet 提供：

- 触发条件
- PhaseBrief
- allowed / forbidden reveal
- effect

它不是死对话树，而是剧情事件池。
